**Conventions for this directory:**

- Filename: `NNNN-kebab-case-title.md` with monotone four-digit numbering. Numbers never reuse, even if an ADR is deleted.
- Status flow: `Proposed → Accepted → Deprecated | Superseded by ADR-NNNN`. Accepted ADRs are never edited; reversing one means writing a new ADR that explicitly supersedes it.
- Length: half a page typical, one page maximum. ADRs are decision anchors, not spec text.
- Cross-link to `docs/design/DESIGN.md` by section number rather than restating the rationale. The design doc is canonical; the ADR is just the lock.
